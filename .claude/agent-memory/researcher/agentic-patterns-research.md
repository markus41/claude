## 21 Core Agentic Design Patterns

### Part One: Foundational Patterns (7 patterns)

#### 1. **Prompt Chaining**
- **What**: Breaking complex tasks into sequential LLM calls where output from one becomes input for the next
- **How**: Linear workflow with clear interfaces between steps
- **Benefits**: 15.6% better accuracy than monolithic prompts, reduced hallucination, modular design
- **Key aspects**:
  - Clear I/O contracts between chain steps
  - Error handling and fallback mechanisms
  - Context management across sequences
  - Intermediate validation before proceeding
  - Logging for debugging
- **Variants**: Sequential chaining (linear), Parallel chaining (concurrent independent steps)
- **Applications**: Content creation pipelines, data processing, QA cycles, code generation, customer service routing
- **Implementation**: Sequential function calls, LangChain, LangGraph, Crew AI, Google ADK
- **Common pitfalls**: Overly complex chains, poor context management, insufficient error handling, ignoring latency/cost

#### 2. **Routing**
- **What**: Classifying inputs and directing them to appropriate handlers, specialized paths, or downstream tasks
- **How**: Initial LLM acts as router, classifying user input intent and selecting best handler
- **Benefits**: Separation of concerns, ability to optimize individual downstream tasks
- **Key aspects**: Intent classification, dynamic path selection, specialized handlers
- **Use case**: Handling diverse queries with different processing requirements

#### 3. **Parallelization**
- **What**: Executing multiple independent tasks simultaneously rather than sequentially
- **How**: Concurrent execution using controlled worker pools
- **Benefits**: Faster processing, multi-perspective analysis leveraging parallel computation
- **Key aspects**: Independent task execution, result aggregation
- **Common patterns**: Voting mechanisms, multi-perspective synthesis

#### 4. **Reflection**
- **What**: Agent evaluating and improving its own outputs through self-assessment
- **How**:
  - Generate initial response
  - Explicitly switch to "critic mode"
  - Assess accuracy, constraint adherence, logical consistency
  - Iterate/refine based on assessment
- **Benefits**: Self-correction, quality improvement, error detection
- **Key aspects**: Self-evaluation layer, iterative refinement

#### 5. **Tool Use (Function Calling)**
- **What**: Agents calling external functions, APIs, tools, or systems to extend capabilities
- **How**: Agent invokes external tools for information retrieval or action execution
- **Benefits**: Access to real-time data, computation, external systems
- **Key aspects**: Function declarations, tool selection, error handling

#### 6. **Planning**
- **What**: Agents creating structured plans or decomposing complex objectives into intermediate steps
- **How**:
  - Break goal into subtasks
  - Create task sequences
  - Plan intermediate steps
  - Execute with validation
- **Benefits**: Efficient multi-step reasoning, reduced computational overhead
- **Key aspects**: Goal decomposition, task prioritization, structured workflows

#### 7. **Multi-Agent Collaboration**
- **What**: Multiple specialized agents working together to solve problems requiring diverse expertise
- **How**:
  - Coordinator/orchestrator manages workflow
  - Specialized agents handle domain-specific tasks
  - Results synthesized/validated
- **Benefits**: Parallel work streams, diverse expertise, quality assurance through critique
- **Key aspects**: Agent specialization, coordination, result synthesis
- **Example**: Security auditor agent reviewing code generated by developer agent

### Part Two: Advanced Systems (4 patterns)

#### 8. **Memory Management**
- **What**: Preserving and retrieving information across sessions and agent interactions
- **How**:
  - Scratchpad accumulates thought history and observations
  - Summarization before feeding back to avoid context bloat
  - External memory stores for essential information
  - Spawning subagents with clean context while maintaining continuity
- **Key aspects**: Context preservation, summarization, handoffs, long-session management
- **Problem solved**: Token limits, context bloat, continuity across long operations

#### 9. **Learning and Adaptation**
- **What**: Agents improving performance over time based on experience
- **How**: Feedback loops, error analysis, pattern recognition
- **Benefits**: Continuous improvement, better handling of edge cases
- **Key aspects**: Feedback integration, pattern learning

#### 10. **Model Context Protocol (MCP)**
- **What**: Standardized framework for integrating tools and data sources with agents
- **Architecture**: Client-host-server model
  - Client: AI agent/application needing information
  - Server: Lightweight connector exposing data source/service
- **How it works**:
  - Multiple MCP servers (one per repo/database/API/tool)
  - Agents connect via unified protocol
  - Dynamic tool discovery
  - Composable patterns
- **Key capabilities**:
  - Dynamic discovery of available tools/services
  - Automatic agent detection of new MCP servers without custom coding
  - Standardized communication across diverse integrations
  - Transport protocols: HTTP, SSE (deprecated), Stdio
- **Integration**: PydanticAI, OpenAI SDK, LangGraph, CrewAI
- **Example**: Agent as MCP client connecting to multiple data sources

#### 11. **Goal Setting and Monitoring**
- **What**: Establishing clear objectives and tracking progress toward goals
- **How**: Setting metrics, monitoring progress, adjusting execution
- **Benefits**: Focused behavior, measurable outcomes
- **Key aspects**: Objective definition, metric tracking, progress monitoring

### Part Three: Production Concerns (3 patterns)

#### 12. **Exception Handling and Recovery**
- **What**: Managing errors and failures gracefully without cascading failures
- **How**:
  - Error detection and logging
  - Fallback mechanisms
  - Graceful degradation
  - Recovery strategies
- **Key aspects**: Error boundaries, recovery paths, logging
- **Challenge addressed**: "What if the agent gets stuck mid-task?"

#### 13. **Human-in-the-Loop**
- **What**: Incorporating human oversight and decision-making into agent execution
- **How**:
  - Escalation triggers for non-delegable decisions
  - Human approval checkpoints
  - Balanced automation vs. control
  - Continuous monitoring and refinement
- **Key aspects**: Decision escalation, approval workflows, human oversight
- **Philosophy**: Some decisions remain non-delegable to AI; agents must hand steering wheel back to humans at key points
- **Implementation**: Guardrails with approval flows, real-time monitoring

#### 14. **Knowledge Retrieval (RAG)**
- **What**: Accessing external knowledge sources to augment agent reasoning
- **How**:
  - Vector databases for document retrieval
  - GraphRAG approaches with knowledge graphs
  - Agentic RAG with multiple retrieval sources
- **Advanced**: GraphRAG stores extracted entities/relationships in knowledge graph for enhanced reasoning over structured domains
- **Benefits**: Grounded responses, access to external data, domain-specific knowledge
- **Key aspects**: Source selection, retrieval strategy, knowledge integration

### Part Four: Multi-Agent Architectures (7 patterns)

#### 15. **Inter-Agent Communication (A2A)**
- **What**: Enabling agents to communicate and collaborate with each other
- **Protocol**: Agent2Agent (A2A) - open protocol by Google (introduced April 2025)
- **Architecture**:
  - A2A Server: Exposes endpoints for discovery and message handling
  - A2A Client: Initiates communication with remote agents
  - Agent Cards: JSON documents at /.well-known/agent-card.json describing identity, capabilities, skills
- **Communication flow**:
  - Discovery (Agent Cards)
  - Initiation (capability-based routing)
  - Completion (message exchange)
- **Protocol details**:
  - JSON-RPC 2.0 over HTTP(S)
  - Synchronous request/response
  - Streaming (SSE)
  - Asynchronous push notifications
- **Key feature**: Agents collaborate without sharing internal memory or proprietary logic (opacity preserved)
- **Use case**: Enterprise workflows, peer-to-peer task outsourcing, specialized agent coordination
- **Pattern**: LLM-driven routing (model decides which agents to invoke based on query)

#### 16. **Resource-Aware Optimization**
- **What**: Efficient use of computational resources in multi-agent systems
- **Challenge**: "Multi-agent systems are expensive. Every sub-agent is burning tokens."
- **Strategy**: Start simple, add complexity only when evidence supports it
- **Warning**: 25x cost increase from unnecessary multi-agent approaches
- **Key aspects**: Resource allocation, cost optimization, efficiency tracking

#### 17. **Reasoning Techniques**
- **What**: Advanced problem-solving methodologies for complex agent tasks
- **Approaches**:
  - Chain-of-Thought reasoning
  - Multi-step reasoning with decomposition
  - Collaborative critique and refinement
- **Benefits**: Better handling of complex problems, improved accuracy
- **Key aspects**: Step-by-step breakdown, verification, refinement

#### 18. **Guardrails and Safety Patterns**
- **What**: Implementing safety constraints and preventing harmful agent behaviors
- **Layered approach**: Input sanitization + policy models + output filters + human review
- **Pattern**: No single guardrail is sufficient
- **Four major guardrail types**:
  - Pre-tool policy checks
  - Drift and failure anomaly detection
  - Graceful fallback mechanisms
  - Human-in-the-loop escalation
- **Real-time guardrails**: Continuous monitoring and refinement
- **Recommended approach**:
  - Ground agents with retrieval
  - Use reasoning-capable models for complex tasks
  - Constrain outputs to typed schemas
  - Run offline evals on realistic scenarios before production
- **Key aspects**: Input validation, policy enforcement, output filtering, human oversight

#### 19. **Evaluation and Monitoring**
- **What**: Assessing performance and system health of agentic systems
- **How**: Continuous monitoring, health checks, performance metrics
- **Benefits**: Visibility into system behavior, early detection of issues
- **Key aspects**: Metric definition, continuous assessment, health tracking
- **Pages**: 18 pages in book (Chapter 19)

#### 20. **Prioritization**
- **What**: Managing task priority and resource allocation across agents
- **How**: Task ranking, priority queues, resource allocation based on priority
- **Benefits**: Efficient task execution, SLA compliance
- **Key aspects**: Priority determination, resource allocation, fairness

#### 21. **Exploration and Discovery**
- **What**: Finding new solutions, approaches, and capabilities
- **How**: Systematic exploration, novelty detection, capability discovery
- **Benefits**: Adaptability, innovation, continuous improvement
- **Key aspects**: Search strategies, evaluation of new approaches

## Frameworks and Implementation Tools

### Major Frameworks Compared

| Framework | Approach | Strengths | Use Case |
|-----------|----------|----------|----------|
| OpenAI Agents SDK | Handoff-based networks | Built-in tracing | LLM-first development |
| LangGraph | Graph-based state mgmt | Superior state management, parallel execution | Complex workflows |

### Framework Capabilities

**Composition**: All major frameworks implement Anthropic's Building Effective Agents patterns in a composable way, allowing developers to chain patterns together.

**MCP Integration**: Modern frameworks support Model Context Protocol for standardized tool/data source integration.

## Key Implementation Concepts

### Andrew Ng's Core Framework

Four foundational capabilities:
1. **Reflection** - Self-evaluation and iterative improvement
2. **Tool Use** - External function integration
3. **Planning** - Task decomposition
4. **Multi-Agent Collaboration** - Specialized agent networks

### Anthropic's Taxonomy

Distinguishes between:
- **Workflows**: Developer-controlled, predetermined paths
- **Agents**: LLM-directed, dynamic decision-making

**Workflow patterns**:
- Prompt chaining
- Routing
- Parallelization
- Orchestrator-workers
- Evaluator-optimizer

### State Machine Loop Architecture

The fundamental agent loop:
```
Perception → Reasoning → Action → Observation → (repeat)
```

## Cost and Performance Considerations

**Critical insight**: Multi-agent systems have significant cost implications. Every sub-agent consumes tokens.

**Recommendation**: Start simple with single-agent patterns, add multi-agent complexity only when data supports it.

**Avoid**: Unnecessary multi-agent approaches (can increase costs 25x).
