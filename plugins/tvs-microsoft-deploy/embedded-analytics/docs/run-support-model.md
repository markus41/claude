# Run & Support Model

## Service Lanes

- **Platform Operations**: workspace administration, refresh reliability, capacity optimization.
- **Application Operations**: embed API uptime, Power Pages integration health, SSO issues.
- **Data Operations**: pipeline failures, quality incidents, schema evolution controls.

## Incident Management

- Severity matrix with response targets by tier.
- Pager rotation for P1/P2 incidents.
- Post-incident review within 3 business days for Sev1.

## Change Management

- Weekly CAB for non-emergency releases.
- Emergency patch lane with rollback checkpoints.
- Monthly dependency and credential rotation review.

## Reporting Cadence

- Weekly ops summary (availability, incidents, trends).
- Monthly service review (SLA, roadmap, risk register).
- Quarterly architecture posture review.
