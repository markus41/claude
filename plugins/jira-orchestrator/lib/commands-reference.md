## Jira Commands

### /jira triage

Triage and analyze a Jira issue for work planning.

```bash
/jira triage <issue-key> [flags]
```

**Flags:**
| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--depth` | `-d` | Analysis depth (quick/standard/deep) | standard |
| `--include-children` | `-c` | Include child issues in analysis | false |
| `--include-links` | `-l` | Analyze linked issues | false |
| `--output` | `-o` | Output format (text/json/markdown) | text |
| `--save` | `-s` | Save report to file | false |
| `--confluence` | | Create Confluence page with analysis | false |
| `--interactive` | `-i` | Interactive mode with prompts | true |
| `--quiet` | `-q` | Suppress progress output | false |

**Examples:**
```bash
/jira triage PROJ-163
/jira triage PROJ-163 --depth deep --include-children
/jira triage PROJ-163 -d quick -o markdown --save
/jira triage PROJ-163 --confluence --no-interactive
```

### /jira transition

Transition a Jira issue to a new status.

```bash
/jira transition <issue-key> <status> [flags]
```

**Flags:**
| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--comment` | `-m` | Add comment with transition | "" |
| `--assignee` | `-a` | Assign to user (email or account ID) | current |
| `--resolution` | `-r` | Set resolution (for Done status) | "" |
| `--validate-only` | | Check if transition is valid, don't execute | false |
| `--force` | `-f` | Skip confirmation prompt | false |
| `--update-children` | | Transition child issues too | false |
| `--interactive` | `-i` | Interactive mode | true |

**Examples:**
```bash
/jira transition PROJ-163 "In Progress"
/jira transition PROJ-163 Done --resolution Fixed --comment "Completed per PR #42"
/jira transition PROJ-163 "Code Review" -a john@example.com --force
/jira transition PROJ-163 Done --update-children
```

### /jira create

Create a new Jira issue with hierarchy validation.

```bash
/jira create <type> [flags]
```

**Flags:**
| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--project` | `-p` | Project key | $JIRA_DEFAULT_PROJECT |
| `--parent` | | Parent issue key (for hierarchy) | "" |
| `--summary` | `-s` | Issue summary | (prompted) |
| `--description` | `-d` | Issue description | (prompted) |
| `--assignee` | `-a` | Assignee | unassigned |
| `--labels` | `-l` | Comma-separated labels | "" |
| `--priority` | | Priority (Highest/High/Medium/Low/Lowest) | Medium |
| `--sprint` | | Sprint name or ID | "" |
| `--interactive` | `-i` | Interactive mode with prompts | true |
| `--dry-run` | | Show what would be created | false |

**Examples:**
```bash
/jira create Story --parent PROJ-100 --summary "Add login feature"
/jira create Task -p PROJ --parent PROJ-101 -s "Implement JWT validation" -a me
/jira create Epic --summary "User Authentication" --labels auth,security
/jira create Subtask --parent PROJ-102 --interactive
```

### /jira search

Search Jira issues with JQL.

```bash
/jira search [query] [flags]
```

**Flags:**
| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--jql` | `-j` | Full JQL query | "" |
| `--project` | `-p` | Filter by project | $JIRA_DEFAULT_PROJECT |
| `--status` | `-s` | Filter by status | "" |
| `--assignee` | `-a` | Filter by assignee (use 'me' for self) | "" |
| `--type` | `-t` | Filter by issue type | "" |
| `--sprint` | | Filter by sprint (current/backlog/name) | "" |
| `--limit` | `-n` | Max results | 50 |
| `--output` | `-o` | Output format (table/json/keys) | table |
| `--fields` | `-f` | Fields to display | key,summary,status |

**Examples:**
```bash
/jira search "authentication"
/jira search --jql "project = PROJ AND status = 'In Progress'"
/jira search --assignee me --status "To Do" --sprint current
/jira search -p PROJ -t Story -s "Code Review" -o json
```

## PR Commands

### /pr-fix

Fix issues from PR review comments with council review.

```bash
/pr-fix <pr-url-or-number> [flags]
```

**Flags:**
| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--jira` | `-j` | Link to Jira issue | "" |
| `--severity` | `-s` | Min severity to fix (critical/high/medium/low) | high |
| `--auto-commit` | | Commit fixes automatically | false |
| `--council-size` | `-c` | Number of review agents (5/10/20) | 20 |
| `--skip-tests` | | Skip running tests after fixes | false |
| `--dry-run` | | Show fixes without applying | false |
| `--interactive` | `-i` | Interactive mode | true |
| `--reply-comments` | | Reply to PR comments when fixed | true |

**Examples:**
```bash
/pr-fix 42 --jira PROJ-163
/pr-fix https://github.com/org/repo/pull/42 --severity critical --auto-commit
/pr-fix 42 --council-size 10 --skip-tests
/pr-fix 42 --dry-run
```

## Harness Commands

### /harness-pipeline

Execute and monitor Harness pipelines.

```bash
/harness-pipeline <action> [flags]
```

**Actions:** `trigger`, `status`, `logs`, `list`, `abort`

**Flags:**
| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--pipeline` | `-p` | Pipeline identifier | (prompted) |
| `--branch` | `-b` | Source branch | current |
| `--jira` | `-j` | Link to Jira issue | "" |
| `--execution` | `-e` | Execution ID (for status/logs) | "" |
| `--stage` | `-s` | Stage name (for logs) | "" |
| `--wait` | `-w` | Wait for completion | false |
| `--timeout` | | Timeout in minutes | 30 |
| `--notify` | `-n` | Send notification on complete | false |
| `--interactive` | `-i` | Interactive mode | true |

**Examples:**
```bash
/harness-pipeline trigger --pipeline ci-build --branch feature/PROJ-163 --jira PROJ-163
/harness-pipeline status --execution abc123
/harness-pipeline logs --execution abc123 --stage build
/harness-pipeline trigger -p deploy-prod --wait --timeout 60
/harness-pipeline list --limit 10
```

## Council Commands

### /council review

Run multi-agent council review on code or PR.

```bash
/council review [target] [flags]
```

**Flags:**
| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--target` | `-t` | Review target (pr/files/diff) | diff |
| `--pr` | `-p` | PR number (if target=pr) | "" |
| `--files` | `-f` | Files to review (glob pattern) | "" |
| `--agents` | `-a` | Number of agents (5/10/20/all) | 20 |
| `--focus` | | Focus area (security/perf/quality/all) | all |
| `--output` | `-o` | Output format (text/json/markdown) | text |
| `--post-pr` | | Post results to PR | false |
| `--jira` | `-j` | Link results to Jira | "" |

**Examples:**
```bash
/council review --target pr --pr 42 --agents 20
/council review --files "src/**/*.ts" --focus security
/council review --target diff --agents 10 --post-pr
/council review -t pr -p 42 -a all --jira PROJ-163
```

| `--expected-type` | `-t` | Expected issue type | "" |

**Examples:**
